What is in a Study File

A study file contains: a name, author, date, description, WRESL files for describing the system (model) and its operating rules, lookup tables (text input data files), State Variable or SV file (.dss input data file), decision varaible or DV file (.dss output data file), initial conditions or Init file (.dss input data file), and the options (settings) for the study.

In general each model study has it’s own directory structure. The study name is normally the base directory in which a "dss" and "run" directory are created. The "dss" directory contains the input and output HEC-DSS timeseries files. The "run" directory, as it’s name implies, is the location where the model is run. The main WRESL file and diagnostic output is located here. Directories named "system", "lookup", and "external" are created under the "run" directory. The "system" directory contains the system description tables and their generated WRESL files. The "lookup" directory contains all relational lookup tables (*.table) required for the model run. The "external" directory contains any external functions (Fortran 90 object files or C++ dynamic link libraries) that are called from the WRESL statements.
