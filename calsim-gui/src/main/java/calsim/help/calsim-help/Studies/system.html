System Tables

Click the "Study" tab then the "System" tab. There are seven tables (Connectivity, Reservoir, Channel, Delivery, Return, Inflow and Weight) under this tab that describe the system configuration and basic operational constraints. Each type of table can be viewed, created or edited, and saved using the File and Edit menus directly above each tab's table. Click File | Load to open each of the seven file types; all end with the .csv extension. These files are typically found in the Run/System directory. The figure below displays a connectivity file. Only a file named correctly can be loaded into a table. To name a file correctly, it must contain the file type (tab title) in its name. For example, a connectivity file must be of the following format: *connectivity*.csv, where the asterisks represent any combination of characters (within the file name size limit of course).

A loaded file can be altered by clicking Edit | Add Row, Insert Row, and/or Delete Row and entering proper data for each row added or inserted. Add Row inserts a row at the end of the table; Insert Row inserts a row before each selected row. Delete Row deletes all selected rows. One row is selected by clicking on it; contiguous multiple rows are normally selected by clicking a row and dragging the cursor over other rows while holding down the left mouse button. Non-contiguous multiple rows are normally selected by clicking a row and clicking other rows while holding down the "Ctrl" button.

Once these files are in their final forms, clicking File | Save for each table will save the *csv file and generate the proper WRESL file for the current system. File | Save As can be used to create a .csv file (and matching WRESL file) with a different name to the loaded file.

Once all the system tables have been created they should be saved to the "system" directory. When each table is saved to this directory a WRESL file is currently generated which contains the constraint translation. At this point, a file named "system.wresl" should be created in the "run\system" directory that should include all the previously generated system files. For example the "system.wresl" file will contain statements similar to the following:

     INCLUDE 'inflow-table.wresl'
     INCLUDE 'channel-table.wresl'
     INCLUDE 'delivery-table.wresl'
     INCLUDE 'return-table.wresl'
     INCLUDE 'reservoir-table.wresl'
     INCLUDE 'connectivity-table.wresl'
     INCLUDE 'weight-table.wresl'
